Perceptron¶

O Perceptron um dos mais antigos modelos de aprendizado de maquina. Ele foi desenvolvido nas dcadas de 1950 e 1960 por Frank Rosenblatt. Esse modelo til para resolver problemas de classificao binria linearmente separveis. Sendo assim, para a satisfatria utilizao do Perceptron, seus dados devem ser separados em entradas e suas respectivas sadas, essas que devem se limitar a duas classificaes, e deve haver uma maneira de separar esses dados linearmente.

O modelo se baseia em organizar os dados em um vetor $ x $ para suas entradas e um vetor $ y $ para suas respectivas sadas. E ento, idealizar um vetor $ w $, de mesma dimenso que cada um dos dados de entrada, que seria responsvel por estabelecer pesos para cada um dos dados de entrada, revelando sua importncia para a deciso, ou classificao final correta. Com esse vetor $ w $ ideal, realizaria-se um produto interno entre ele e o vetor $ x $ e revelaria qual a classificao correta, por exemplo:

Classificar como sim se $ \sum \limits _{i=1} ^{d} w_i x_i > constante $

Classificar como no se $ \sum \limits _{i=1} ^{d} w_i x_i < constante $

Ou, compactamente, assumindo um $ h(x) $ como uma funo que da como resultado +1 se sim e -1 se no:

$h(x) = sign((\sum \limits _{i=1} ^{d} w_i x_i) + b)$,

onde o $ b $ o chamado "bias", ou o limiar em relao a que consideramos uma classificao ou outra. Percebe-se que tanto os valores de $ w $ quanto de $ b $ so relativos, e pode haver mais de um conjunto deles que separa corretamente os dados linearmente.

O alguns algoritmos ento surgem para tentar, apenas tendo dados de entrada e sada, aproximar um vetor $ w $ que sirva para resolver o problema da maneira mais prxima da correta possvel. Um algoritmo possvel ser implementado a seguir, ele se baseia em dados de entrada com duas dimenses, e que so completamente separveis por uma reta.

Esse algoritmo se baseia em inicializar o vetor $ w $ com valores aleatrios, realizar o produto interno: $ h(x) = \langle w, x \rangle$ e, em seguida, procurar algum dado que esteja classificado de uma maneira errnea, caso haja, o valor de $ w $ atualizado de uma determinada maneira, e o processo se repete at que todos os pontos estejam classificados corretamente.

Se os dados forem linearmente separveis, e a atualizao do vetor $ w $ ocorrer da maneira a seguir: $ w(t + 1) = w(t) + y(t)x(t)$ possvel mostrar que o algoritmo converge para que todos os pontos estejam separados corretamente.

Algoritmo:

1) Inicializa $ w $ com valores aleatriamente escolhidos

2) Verifica se h algum dado classificado errado

3) Se h algum dado classificado errado, atualiza $ w(t + 1) = w(t) + y(t)x(t)$ e volta para 2)

4) Fim.

import numpy as np 
import matplotlib.pyplot as plt
IT_LIMIT = 100000# Limite de iteraes
def esbocar(w, b, x, y, i):
    pbx = np.max(x[:,0])
    pby = np.max(x[:,1])
    psx = np.min(x[:,0])
    psy = np.min(x[:,1])
    plt.figure(i)
    plt.scatter(x[:,0],x[:,1],c=y)
    u = np.linspace(psx - 1, pbx + 1, 1000)
    v = -1 * (w[0] * u + b)/w[1]
    plt.plot(u,v)
    plt.grid(True)
    plt.xlim(psx - 1, pbx + 1)
    plt.ylim(psy - 1, pby + 1)
    plt.xlabel('x axis')
    plt.ylabel('y axis')
    plt.title("Reta de separacao")
    
def percepetron(x, y, b, wi):
    it = 0 #nmero de iteraes = 0
    w = wi #inicializando com valores aleatrios
    wb = w
    erros = 1
    merros = float("inf")
    esbocar(w, b, x, y, 0)
    while (erros and it < IT_LIMIT):  #Enquanto o Algoritmo encontrar pontos classificados erroneamente, ele continua a rodar, a menos que o nmero de iteraes passe o limite
        erros = 0;
        for i in range(len(x[:,0])):
            s = w.dot(x[i,:])
            #s = w[0]*x[i][0] + w[1]*x[i][1] # s  o resultado do produto interno do vetor w pelo vetor x
            if ( s + b >= 0): # Analiza a classificao
                h = 1;
            else:
                h = -1;
            if (h * y[i] < 0):
                w = w + y[i]*x[i]
                #w[0] = w[0] + x[i][0] * y[i] # Atualizao do vetor w
                #w[1] = w[1] + x[i][1] * y[i]
                erros += 1;
        it += 1
        if erros <= merros:
            wb = w
            merros = erros
        if (it >= IT_LIMIT):
            input("A reta no foi encontrada aps 100000 iteraes")
    esbocar(w, b, x, y, it)
    return wb, it   


def predict(x, w, b):
    s = w.dot(x)
    if (s + b >= 0):
        h = 1;
        print("O ponto recebido  amarelo")
    else:
        h = -1
        print("O ponto recebido  azul")
    return h

Análise¶

Aps alguns testes, possvel afirmar que esse algoritmo extremamente limitado. Apesar de funcionar bem para uma quantidade de pontos pequena, se a quantidade de pontos for maior que 6, e o conjunto vetor $ w_i $ e "bias" iniciais gerarem uma reta muito distante da ideal, o nmero de iteraes j passa do limite estabelecido.

Com isso estabelecido, uma sugesto de melhoria para o algoritmo seria um fator que calcula o erro de cada ponto, que poderia entrar no calculo da atualizao com o objetivo de fazer com cada atualizao aproxime mais a reta atual da reta procurada.
